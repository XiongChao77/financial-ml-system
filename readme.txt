原始数据：

📊 基本价格与时间信息
open_time：K线开始时间（毫秒时间戳）。
open：该周期的开盘价。
high：该周期的最高价。
low：该周期的最低价。
close：该周期的收盘价。
close_time：K线结束时间（毫秒时间戳）。
📈 成交量相关
volume：该周期内的成交量（以交易对的 基础资产 base asset 计量，比如 BTCUSDT 里的 BTC）。
quote_asset_volume：该周期内的成交额（以 计价资产 quote asset 计量，比如 BTCUSDT 里的 USDT）。
公式关系： quote_asset_volume ≈ volume × 平均成交价格
🔄 交易统计
number_of_trades ：该周期内成交的笔数（撮合成功的订单数，不区分买卖方向）。
taker_buy_base_volume ：taker 主动买入的基础资产成交量。
举例：在 BTCUSDT 中，taker 下的买单成交了 5 BTC，就计入这里。
taker_buy_quote_volume：taker 主动买入对应的计价资产成交额。
举例：taker 买了 5 BTC，花了 100,000 USDT，这里就是 100,000。

| 指标                 | 含义                       | 解释                                                 |
| ------------------ | ------------------------ | -------------------------------------------------- |
| **precision（精确率）** | 预测为该类别的样本中，真实属于该类别的比例    | `precision = TP / (TP + FP)`<br>→ 预测的“准”           |
| **recall（召回率）**    | 该类别真实样本中，被模型正确预测出来的比例    | `recall = TP / (TP + FN)`<br>→ 预测的“全”              |
| **f1-score（F1分数）** | precision 与 recall 的调和平均 | $F1 = \frac{2\cdot P \cdot R}{P+R}$<br>→ 平衡“准”和“全” |
| **support**        | 该类别在测试集中的样本数量            | 样本基数，便于看类别规模                                       |

| 名称                     | 含义                                            | 说明                                 |
| ---------------------- | --------------------------------------------- | ---------------------------------- |
| **accuracy（准确率）**      | 全部样本中预测正确的比例                                  | $\frac{\text{所有正确预测}}{\text{总样本}}$ |
| **macro avg（宏平均）**     | 先在每个类别算 precision/recall/F1，再对类别**等权平均**      | 不考虑类别数量差别，适合看模型对小类是否公平             |
| **weighted avg（加权平均）** | 先在每类算 precision/recall/F1，再按 support（样本数）加权平均 | 样本多的类别影响更大，接近整体表现                  |

*******************************数据挖掘*********************************************
因子相关性分析:
    使用归一化之后的数据进行相关性分析。大量的经典文献和现代深度学习在金融领域的应用论文都间接或直接地证实了归一化（或差分、对数收益率转换）是有效衡量因子预测能力的必要前提
    Granger & Newbold (1974) "Spurious regressions in econometrics": 这篇获得诺贝尔奖级别的论文奠定了基础。它证明了如果直接使用非平稳的时间序列（如原始价格）进行回归或相关性分析，即使两个变量完全无关，t统计量也会显示显著相关。结论：必须通过差分或归一化让数据平稳后，统计量的显著性才可信。
    Marcos López de Prado (2018) "Advances in Financial Machine Learning": 它是现代量化ML的圣经。López de Prado 在书中专门有一章讨论 "Fractional Differentiation" (分数阶差分)。他明确指出：标准的机器学习模型无法处理非平稳数据（原始价格），必须将数据转换为平稳序列（类似你的归一化或收益率）才能保留预测能力（Memory）。 他强调，如果输入特征没有经过适当的平稳化处理，相关性分析将毫无意义。

*******************************策略有效性验证*********************************************
1. 滚动窗口分析 (Walk-Forward Analysis) —— 最推荐
既然不能把数据切成两大块，那就切成很多个小块，模拟真实的时间流逝。这在量化中是最标准的做法。

做法：

设定一个训练窗口（例如 6 个月）和一个测试窗口（例如 1 个月）。

Step 1：用第 1-6 月的数据优化 thresh，得出最佳参数（比如 0.5），然后在第 7 个月进行测试。

Step 2：向前滚动！用第 2-7 月的数据重新优化 thresh（这次可能变成 0.55），然后在第 8 个月进行测试。

Step 3：继续滚动...

优点：

动态适应：你的参数是跟着市场变化的，这符合实盘逻辑（你会定期调整策略）。

数据利用率高：所有数据都被既当过训练集，也当过测试集（在不同时间点）。

样本外验证：你永远是在用“过去”的数据推导参数，去跑“未来”的一个月，没有偷看答案。

2. 跨币种验证 (Cross-Sectional Validation)
既然时间不够长，那我们就利用空间（币种数量）来凑。

逻辑：如果你的逻辑（例如：RSI 超卖 + 突破）是捕捉市场情绪的普适规律，那么它不应该只在 BTC 上有效。

做法：

在 ETH 的历史数据上寻找最佳 thresh。

直接把这个参数拿去跑 SOL、BNB 或 BTC 的回测。

判定：如果在 ETH 上拟合出的参数，在 SOL 上跑得一塌糊涂，说明这个参数是“过度拟合”了 ETH 的特定走势，而不是捕捉到了通用的交易逻辑。

注意：要选择相关性没那么高的币种（虽然币圈整体相关性很高），或者选择不同赛道（公链 vs DeFi vs Meme）进行验证。
